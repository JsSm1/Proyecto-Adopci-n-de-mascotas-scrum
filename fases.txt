1. Planificación - Fase 1
1.1 Meta principal: El objetivo principal de este proyecto es el diseño, desarrollo e implementación de una plataforma web intuitiva y centralizada que permita a los usuarios visualizar y filtrar un catálogo exclusivo de perros y gatos en adopción. Asimismo, el sistema integrará un canal seguro para la recaudación de donaciones voluntarias, para la sostenibilidad del sitio.

1.2 Análisis de viabilidad:
	 Viabilidad económica:
Costes: 
- Inversión en diseño de la plataforma, horas de desarrollo.
 - costos fijos mensuales de mantenimiento (Hosting, base de datos)
Beneficios:
Monetario: Ingreso continuo de dinero a través del módulo de donaciones voluntarias para cubrir alimento y veterinario.
Operativo: Reducción de costes en publicidad impresa, ya que todo el catálogo estará digitalizado.
 Viabilidad técnica:
Se determina que el proyecto  es técnicamente posible utilizando tecnologías como:Html,CSS  React, Java(Springboot).
El sistema es capaz de almacenar de forma ligera registros de texto (raza, edad, personalidad) y archivos multimedia (fotografías de las mascotas).
La pasarela de pagos se resolverá mediante la integración de una API externa segura, evitando la necesidad de construir infraestructura bancaria desde cero. 

	Viabilidad Operativa:
Usuarios Finales (Adoptantes): Al ser un público sin conocimientos técnicos profundos, la interfaz contará con un diseño minimalista y flujos de navegación sencillos (botones grandes e identificables para "Perros", "Gatos" y "Donar").
Usuarios Administradores (Personal del Refugio): Se creará un panel de control simplificado donde los voluntarios subirán nuevos animales rellenando campos básicos y listas desplegables, sin requerir conocimientos de programación. 

1.3 PLAN DEL PROYECTO DE ALTO NIVEL
El panorama general del proyecto se divide en cuatro hitos macro esenciales para asegurar el cumplimiento de las tareas y objetivos generales:
Hito 1: Arquitectura y Base de Datos
Descripción: Configuración del servidor e instalación de las tablas de datos para el almacenamiento de la información de las mascotas.
Hito 2: Interfaz de Usuario y Filtros
Descripción: Programación visual de la página web, incluyendo el despliegue del catálogo y los filtros exclusivos por especie (perros y gatos).
Hito 3: Pasarela de Donaciones
Descripción: Integración de la API de pagos y habilitación del formulario interactivo para aportes económicos voluntarios.
Hito 4: Control de Calidad y Lanzamiento
Descripción: Pruebas de velocidad de carga, verificación de seguridad y capacitación del personal del refugio para el uso del sistema.




5. Pruebas  e Integración Fase 5.
5.1 Prueba de los programas y procedimientos
El profesional de AC ejecutará estrategias de control tanto manuales como automatizadas para validar el comportamiento del sistema desarrollado:
Verificación del Catálogo: Comprobación de que el filtrado funciona con precisión quirúrgica, asegurando que al seleccionar "Gatos" no se visualicen perros y viceversa.
Validación de Fichas de Mascotas: Inspección de las tarjetas informativas para verificar que la foto de la mascota, la raza, la edad y su descripción de personalidad se desplieguen sin errores visuales en celulares y computadoras.
Pruebas de Integración y Regresión: Uso de herramientas técnicas como Postman para interactuar con las APIs del backend, certificando que los datos de las donaciones voluntarias se registren de forma íntegra en la base de datos y que el código nuevo no altere las funciones ya existentes.
5.2. Determinar si el diseño propuesto cumple con la meta inicial de la fase de planificación 
Cumplimiento de la Meta: Se evalúa si el sistema es lo suficientemente intuitivo y sencillo para que adoptantes y donantes sin conocimientos informáticos realicen transacciones o busquen un animal en pocos clics.
Optimización del Refugio: Se determina si la carga del catálogo digital cumple con el objetivo de reducir el flujo de trabajo manual del personal del refugio al automatizar la visualización de los animales rescatados.
5.3 Roles involucrados
Analista de Pruebas: Responsable de diseñar los escenarios de prueba, establecer los criterios de aceptación y redactar las guías de validación funcional del sistema de adopción y donaciones.
Tester: Encargado de ejecutar de manera práctica los flujos del sitio en busca de fallas o enlaces caídos, apoyándose en Postman para corroborar que el servidor responda de forma segura y correcta ante las solicitudes de los usuarios.
Desarrollador: Recibe los informes detallados de fallas (bugs) emitidos por el Tester, localiza la línea de código afectada para su inmediata corrección y vuelve a probar la ruta mediante simulaciones técnicas antes de entregar el incremento limpio.





🔧 6. Mantenimiento  - Fase 6
6.1 Proporcionar asistencia continua
Soporte Técnico Operativo: Resolución inmediata si se llega a presentar una caída o interrupción en el procesamiento de las donaciones voluntarias de los usuarios.
6.2 Ajustes para las necesidades cambiantes del cliente
El software se adapta a la evolución y crecimiento del refugio, respondiendo a requerimientos que el cliente no anticipó al inicio del proyecto:
Si el refugio decide cambiar la pasarela de pagos (por ejemplo, migrar de PayPal a Mercado Pago) o modificar el orden de prioridad del catálogo de mascotas.
Ajustar las preguntas del cuestionario de adopción si el personal del refugio detecta que necesita filtrar mejor el espacio de vivienda de los solicitantes.

	6.3. Roles implicados en esta fase
El equipo mínimo requerido para asegurar la continuidad operativa y la vigencia tecnológica de la web se distribuye de la siguiente manera:
Desarrollador: Encargado técnico de aplicar los parches de seguridad, alterar las líneas de código para realizar los ajustes solicitados por el cliente y reconfigurar las integraciones de software cuando sea necesario.
Tester: Responsable de verificar en un entorno seguro que las actualizaciones del programador solucionen los incidentes reportados y que los nuevos ajustes funcionales no rompan el catálogo existente antes de subirlos a producción.
